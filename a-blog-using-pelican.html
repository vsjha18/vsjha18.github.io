A Blog Using Pelican !

Mon 26 January 2015 python pelican blogging

From a long time I was looking for a blogging solution which is not just hassle free to manage but also cheap and easy to host. At the same time I was is no mood to trade the configurability aspect for the above features. And off course who wants to learn a new technology just becuase he wants to maintain a blog and on the same lines I wished that all of it I could get in python.

I explored several options, and went through many articles which had montonous titles like "5 best blogging framework ....". Out of all this exercise two choices stood out and they were the giants of this domain, Wordpress and Blogger.

Though they are very good solutions for the problem but somehow did't match my taste.

The Wordpress

Without any doubt this is the most popular blogging framework and probably the most feature rich also. But the main reason behind it's popularity is the rich admin console which can be used for almost all corners of the blog (not quite the way I would like). This benfit brings a lot of bloggers who are not essentially engineers on the Wordpress boat.

I hunted down this option because of:

  • Too many features which I actually didn't need. I wanted to keep it just as simple as possible.
  • Configuring it would require me to learn PHP, which doesn't find a place in my technical roadmap.
  • Hosting requires more money than I would have wished.

Blogger

This is also a nice framework and I used it for quite a while for running a puzzle blog. It was far simpler than wordpress which I consider a plus but it was somewhat boring as well.

Very few templates were available and hosnestly almost all were bad. Making any serious change required you to learn the Google APIs and messing up with HTML for UI tweaks was also not the easiest thing to do.

It felt to me that even google was not very serious about it becuase it still has the same feature and templates which were there around five years ago.

Static Site Generators

This new concept really excited me when I heard about Jekyll through Github. It was really somethig which I was looking for. I can write on the system on any plain text editor. And there were many templates to chose from.

But there was on problem. It's not really a Jekyl problem but it was related to me. I did't know Ruby and for making any changes of higher order would require some Ruby knowledge. Though it's a nice language to learn (unlike PHP) but my priorities were not allowing me to go into that direction. I was wondering whether is there any Jekyl like stuff available in Python.

And Finally the Pelican

After some google search, Pelican was not difficult to find. I used almost similar technologies which I already knew, like Jinja2 teamplates. It is really easy to tweak the templates and design one from scratch if you know Jinja2 well.

Since it was a static site generator hence it didn't require any database setups.

I always considered databases to be an overkill for simple applications like blog !

Pelican didn't take much time to impress me to an extent that I started developing my own theme. I got an amazing starting point which is pelican-bootstrap3. If you know bootstrap then this project can prove to be a very good starting point. The theme supported all the fancy stuff like.

  • Google Analytics.
  • Disqus for comment handling.
  • AddThis - For adding social buttons.

Github + Pelican a Killer Combo

I am using Github to host the blog which you are reading right now. It's a very easy to use it you work with git (All of us do). And you dont have to pay anthing for this. It is free !! All you need to pay if for domain and that too is not mandatory as long as you are happy with the gihub sub-domain.
