A Blog Using Pelican !

Mon 02 March 2015 python pelican blogging

From a long time I was looking for a blogging solution which is not just hassle free to manage but also cheap and easy to host. At the same time I was is no mood to trade the configurability aspect for the above features. And also I didn't want to learn any new technology just for the sake of maintiaing a blog.

I explored several options, and went through many articles which had monotonous titles like "5 best blogging framework ....". Out of all this exercise two obvious choices stood out, viz. Wordpress and Blogger.

Though they are very good solutions for the problem but somehow it did't match my taste.

The Wordpress

Beyond doubt this is the most popular blogging framework and probably the most feature rich also. But the main reason behind it's popularity is the strong community and the rich admin console which can be used to control most of the aspects of the site. And this was the big reason for wide adoption of Wordpress becuase you really don't need to be techie to own a blog. People with limited computer knowledge were able to manage and host their own blog.

I hunted down this option because of:

  • Too many features which I actually didn't need. I wanted to keep it just as simple as possible.
  • Configuring it would require me to learn PHP, which doesn't find a place in my technical roadmap. (at least after learning python!)
  • Hosting requires more money than I was ready to spend. (I told you it has to be super cheap)
  • Theaming also requires to understand the Worpress jargon which is overwhelming for me.

Blogger

This is also a nice framework and I used it for quite a while for running a puzzle blog. It was far simpler than wordpress which I consider a plus but it was somewhat boring as well.

If you see blogger in last 5 years, nothing much has changed, this is very unlike all other Google products. Looks at it only feels that Google is using it as just one more medium to push their advertisements.

Very few templates to choose from and look and feel of all of them reminds me of the web of 2005 era. Most of the widgets are only there for the sake of increasing the number of widgets, very few are useful.

Static Site Generators

This new concept really excited me when I heard about Jekyll through Github. It was really somethig which I was looking for. I can write articles on any plain text editor, which was a great plus. And there were many templates to chose from. Being static site it only had features which a static site can afford to have without an active application server.

But there was a problem. It's not really a Jekyll issue, but rather a matter of personal taste. I did't know Ruby and for making any changes of higher order would require some Ruby knowledge. Though it's a nice language to learn (unlike PHP) but my priorities were not allowing me to go into that direction. I was just wondering whether is there anything like PyJekyll !!

And Finally the Pelican

The beauty of python is, you name it and you have it. And a static site generator was no exception either. It only took few google searches to reach Pelican. And it had almost everything which I wished for. It hardly took anyt ime to impress me to a point of no return. At a glance:

  • Had almost all essential features which a blog should have. (and did't have ones which I consider overkill)
  • Templating was super easy because of Jinja2 templating system.
  • I could write in rst format. I am more fluent in rst compared to markdown.
  • I don't have to write in a WYSIWYG editor, comfort of vim was there to stay.
  • Integration with almost all the cloud service out of box.
  • Things like google analytics, disqus were present out of box.
  • One of the best project documentations.

The Theme Hunt

Next problem to solve was the theme hunt. I always wanted to do it myself rather than using a readymade theme (mostly for learning sake). One thing I must say that becuase of the relatively smaller community than the Jekyll, the number of available themes (which are worth using also) were less. But I found an amazing theme which I could use as a great starting point. Pelican-Bootstrap3 is an awsome project. It is a Bootstrap based theme (one more keyword which I am comfortable at) and does just enough to make a great starting point. That doesn't mean it is not directly usable, in fact it looks pretty descent. It supports most of the Social stuff and cloud services. To name a few:

  • Google Analytics.
  • Disqus for comment handling.
  • AddThis - For adding social buttons.

I modified that theme and created a new one and this site is currently using that.

Github + Pelican a Killer Combo

Since it is a Pelican produces static sites hence you really don't require any heavy server running your blog. A basic static file server with modest capabilities is just more enough to do the job. This blog is hosted on the github which is totally free. That also means that apart from paying for purchasing a domain (that too is not mandatory) you don't need to spend a single penny.

So the total hosting cost for me was Rs 300 (approximately 3 USD), which is less that a cost for one Mc Donald Meal.

The Current Theme

This theme is a fork of Pelican-Bootstrap3 project. I corrected some markups and styled it a bit. This is still a work in progress but it is completely usable.

click here to visit the project page. You can clone it as follows

You can clone it like this:

git clone https://github.com/vsjha18/pelican-elegant.git

Make sure you have following lines in the pelicanconf.py files.

DISPLAY_RECENT_POSTS_ON_SIDEBAR=True
DISPLAY_CATEGORIES_ON_SIDEBAR=True
DISPLAY_CATEGORIES_ON_MENU=True
BOOTSTRAP_THEME = 'cosmo'
BOOTSTRAP_NAVBAR_INVERSE=True

For rest of the things you can follow steps on the project page and and Pelican manual.
